Tổng quan hệ thống

Về bản chất, app của bạn sẽ xin quyền từ tài khoản Google, lấy token, rồi dùng token đó gọi API để đọc hoặc cập nhật tài liệu.

Google Docs API

Dùng để đọc cấu trúc tài liệu, chèn văn bản, replace nội dung, format heading, table và cập nhật nội dung qua batchUpdate.

Google Drive API

Dùng để tìm file, đọc metadata, tạo file mới, quản lý quyền truy cập và hỗ trợ thao tác quanh tài liệu Google Docs.

OAuth 2.0

Đây là lớp xác thực bắt buộc. Người dùng đăng nhập Google, bấm Allow, rồi app lấy access token và refresh token để gọi API hợp lệ.

Các bước thiết lập

Làm theo thứ tự này là ổn nhất nếu bạn muốn sau này để agent hoặc app nội bộ ghi Google Docs tự động.

1

Tạo Google Cloud Project

Vào Google Cloud Console, tạo một project mới để gom toàn bộ cấu hình API, OAuth và quota vào cùng một nơi.

2

Bật Google Docs API và Google Drive API

Bạn cần bật cả hai. Docs API để sửa nội dung; Drive API để xử lý file, metadata và quyền truy cập.

3

Tạo OAuth Consent Screen

Nhập tên ứng dụng, email hỗ trợ, thêm scope cần dùng và cấu hình màn hình xin quyền từ Google.

4

Tạo OAuth Client ID

Chọn loại ứng dụng phù hợp: Desktop App nếu chạy local, hoặc Web App nếu có callback URL.

5

Người dùng đăng nhập và bấm Allow

Ứng dụng mở trình duyệt, người dùng đăng nhập Google, sau đó cấp quyền để app nhận access token và refresh token.

6

Dùng token để gọi API

Access token dùng để gọi Docs API / Drive API. Refresh token dùng để xin access token mới khi token cũ hết hạn.

Scope và endpoint quan trọng

Bạn không cần xin quyền quá rộng nếu nhu cầu chỉ là đọc/sửa docs.

Scope nên dùng

  • https://www.googleapis.com/auth/documents
  • https://www.googleapis.com/auth/drive.file
  • Nếu cần mạnh hơn: https://www.googleapis.com/auth/drive

Đọc tài liệu

GET https://docs.googleapis.com/v1/documents/{documentId}

Dùng để lấy cấu trúc document, nội dung text và vị trí index.

Cập nhật tài liệu

POST https://docs.googleapis.com/v1/documents/{documentId}:batchUpdate

Dùng để insert text, replace text, format paragraph, heading, list, table...

Ví dụ request để chèn text vào Google Docs

Đây là dạng body phổ biến nhất khi bạn muốn thêm nội dung mới vào đầu tài liệu.

{
  "requests": [
    {
      "insertText": {
        "location": {
          "index": 1
        },
        "text": "Hello Ngọc Tiến"
      }
    }
  ]
}
POST https://docs.googleapis.com/v1/documents/{documentId}:batchUpdate
Authorization: Bearer ACCESS_TOKEN
Content-Type: application/json

Nếu muốn assistant tự ghi Docs lâu dài

Có 2 hướng phổ biến và thực tế nhất.

Hướng A — Viết bridge nội bộ

Dùng Node.js hoặc Python để login OAuth 1 lần, lưu refresh token an toàn, sau đó tạo một API nội bộ để append/replace text trong docs.

Hướng B — Dùng MCP / connector

Nếu hệ thống trợ lý hỗ trợ tool/connector, bạn có thể tạo một Google Docs connector với các lệnh như read_doc, append_doc, replace_text.

Hướng C — Chạy script cục bộ

Đơn giản nhất để thử nhanh: tạo OAuth app, lấy token, rồi viết script append nội dung vào một document cụ thể.

Tóm tắt ngắn

  • Google Docs cá nhân không thể sửa chỉ bằng API key đơn lẻ.
  • Bạn cần Google Cloud Project + Google Docs API + Google Drive API.
  • Xác thực bắt buộc là OAuth 2.0.
  • Sau khi người dùng bấm Allow, ứng dụng lấy access token / refresh token.
  • Dùng batchUpdate của Docs API để chèn hoặc sửa nội dung.
Nếu muốn, bước tiếp theo mình có thể làm cho bạn là viết luôn một script Node.js để đăng nhập Google và tự động ghi nội dung vào một Google Docs cụ thể.
